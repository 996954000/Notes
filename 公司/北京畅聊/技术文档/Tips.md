# 数据配置关联问题（世界当前状态配置与数据耦合）
初步配置表中，通过初始化了一个任务等级对应属性的配置表，并通过得到升级信息后直接从当前等级信息配置表中进行数据抓取，并将数据套用在人物上。
```
// 人物等级配置
config = [
    lv = 1...
],[
    lv = 2...
]...

// 当前“世界”状态(原思路类似：当世界状态为1级时对应人物1级)，类似于：
worldLevel = [
    lv = 1,
    config[Lv 1] // 当前世界状态对应的任务状态
],[
    lv = 2,
    config[lv 2]
]

// 当前“其他使用到lv配置的数据”
otherDataConfig = [
    name = "lv one",
    config[Lv 1]
],[
    name = "lv two",
    config[Lv 2]
],


```
这种写法显然的将人物等级数据与状态数据有了严重的耦合，当其他地方也在使用改类数据时，便出现信息不同步的问题

`当前想到的解决方法便是将数据与行为分离，将升级行为与等级数据进行分离，升级行为暴露方法，等级数据提供对应等级数据`
