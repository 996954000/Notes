# 场景
场景主要元素有
## 建筑
  `各类功能建筑，无功能建筑等，包括城墙，兵营，经济产出建筑，兵种升级建筑等，`
- 类别：
  - "防御建筑"：`炮塔等防御建筑，其制作更贴近于友方角色的设计，具有各类属性以及状态，代码编写时应该是放在同一父类下，将其移动速度置0，隐藏不必要的属性并添加部分特有属性即可`
  - 兵营：作为兵种的出生点坐标，提供兵种升级交互功能
- 建筑交互
  - 建筑自身升级，包括城墙等建筑点击进行升级操作，改变其属性、外观，查表配置更新
  - 建筑功能，如经济产出，士兵产出，交互进入士兵强化菜单或者弃用菜单，单个建筑负责单种角色的简单强化管理
## 角色
  `区分为敌我角色，中立角色等`
- 我方角色
  - 各类兵种：不同的兵种具有不同的行为模式，属性，具体行为通过配置其状态机或行为树实现。
  - 升级：查表配置每一等级的属性数据，外观等，角色的升级仅应用于下一次产出的兵种，即更新下对象池的预制件，并在低级角色全部回收后释放低级资源。
- 敌方角色
  - 各类僵尸：同我方兵种，各类行为模式，行为树，状态机，对象池管理等
  - 不存在同类僵尸升级逻辑
- 通用
  - 状态：
    - 生成（非必要）（可以做僵尸或己方角色的入场动画，）
    - 移动（寻路（从出生点至城墙点位））
    - 待命
    - 攻击
    - 死亡 
    - 巡逻（非必要）
    - 技能释放
  - 配置:
    - 动态属性：移动速度，攻击速度，伤害，血量，防御属性，技能等
    - 静态属性：图鉴？
- 中立角色：商人？

- 关卡优先设计好，静态加载即可

# 角色



以下是使用 Unity 制作一款城墙防御僵尸小游戏的系统设计方案，涵盖核心系统、组件和实现要点，力求简洁清晰，同时为开发提供结构化的指导。

---

### 1. 游戏核心机制
- **目标**：玩家控制城墙上的士兵和炮塔，射击或炮轰不断涌来的僵尸，阻止僵尸突破城墙。
- **玩法**：
  - 士兵和炮塔自动或手动攻击僵尸。
  - 玩家管理资源（例如弹药、金币）升级武器或修复城墙。
  - 僵尸以波次（Wave）形式出现，强度逐渐增加。
  - 游戏失败条件：僵尸突破城墙或城墙耐久度归零。
- **视角**：2D 侧视（类似《植物大战僵尸》）或 2.5D 视角，城墙位于屏幕左侧，僵尸从右侧逼近。

---

### 2. 系统设计模块

#### 2.1 角色系统
- **士兵（Soldier）**：
  - 属性：生命值、攻击力、射速、射程。
  - 行为：自动瞄准最近的僵尸，发射子弹。
  - 控制：玩家可手动切换目标或调整优先级（例如优先攻击血量高的僵尸）。
  - 升级：提升射速、攻击力或解锁特殊弹药（例如爆炸弹）。
- **炮塔（Turret）**：
  - 属性：生命值、攻击力、射速、射程、爆炸范围（可选）。
  - 行为：周期性发射炮弹，造成范围伤害。
  - 升级：增加射程、爆炸范围或特殊效果（例如减速）。
- **僵尸（Zombie）**：
  - 类型：普通僵尸、快速僵尸、坦克僵尸（高血量）。
  - 属性：生命值、移动速度、攻击力。
  - 行为：沿直线向城墙移动，接触城墙后攻击城墙。
  - 特殊能力：某些僵尸可能有护盾、加速或群体增益。
- **城墙（Wall）**：
  - 属性：耐久度。
  - 行为：承受僵尸攻击，耐久度降低至零时游戏失败。
  - 修复：玩家使用资源修复城墙。

#### 2.2 战斗系统
- **射击机制**：
  - 士兵和炮塔通过射线检测（Raycast）或触发器（Trigger）锁定目标。
  - 子弹和炮弹为带有碰撞体的 GameObject，命中僵尸后造成伤害。
  - 炮塔炮弹可附加范围伤害，使用 Physics2D.OverlapCircle 检测范围内的敌人。
- **伤害计算**：
  - 伤害公式：`EnemyHealth -= (WeaponDamage - EnemyArmor) * DamageMultiplier`。
  - 暴击机制：随机触发，增加伤害（例如 1.5 倍）。
- **击杀奖励**：
  - 击杀僵尸掉落金币或资源，用于升级或购买单位。

#### 2.3 波次系统（Wave System）
- **生成逻辑**：
  - 每波僵尸由预设的生成序列控制（例如 10 只普通僵尸 + 2 只坦克僵尸）。
  - 使用协程（Coroutine）控制生成间隔和波次间隔。
  - 波次难度递增：增加僵尸数量、生命值、速度或引入新类型。
- **触发条件**：
  - 当前波次所有僵尸被消灭后，进入短暂准备时间（例如 15 秒），然后开始下一波。
- **UI 显示**：
  - 显示当前波数和剩余敌人数量。

#### 2.4 资源管理系统
- **资源类型**：
  - **金币**：通过击杀僵尸或波次奖励获得。
  - **弹药**：限制士兵和炮塔的攻击次数（可选）。
- **使用**：
  - 购买新士兵或炮塔。
  - 升级单位属性。
  - 修复城墙。
- **UI**：
  - 资源数量实时显示。
  - 按钮提示可用操作（例如“升级”按钮变灰）。

#### 2.5 UI 与交互系统
- **主界面**：
  - 显示城墙耐久、资源、波次信息。
  - 单位管理面板：选择士兵/炮塔、升级或出售。
- **操作**：
  - 点击城墙上的位置放置或选中单位。
  - 拖拽调整单位位置（可选）。
  - 快捷键或按钮触发修复城墙、切换攻击模式。
- **暂停菜单**：
  - 暂停游戏、调整设置、重启或退出。

#### 2.6 音效与视觉效果
- **音效**：
  - 射击音效、爆炸音效、僵尸死亡音效、背景音乐。
  - 使用 AudioSource 组件动态播放。
- **视觉效果**：
  - 子弹轨迹：使用 Particle System 或 Line Renderer。
  - 爆炸效果：Particle System。
  - 僵尸死亡：播放死亡动画后销毁对象。
  - UI 反馈：按钮点击动画、资源变化提示。

### 3. Unity 实现要点

#### 3.1 项目结构
- **场景（Scene）**：
  - 主场景：包含城墙、地面、生成点、UI 画布。
  - 菜单场景：开始界面、设置界面。
- **预制体（Prefab）**：
  - 士兵、炮塔、僵尸、子弹、炮弹。
  - 保存通用属性和行为，方便复用。
- **层级（Hierarchy）**：
  - 根节点：GameManager、UIManager、WaveManager。
  - 子节点：Units（士兵和炮塔）、Enemies（僵尸）、Projectiles（子弹和炮弹）。

#### 3.2 核心脚本
- **GameManager**：
  - 管理游戏状态（进行中、暂停、失败）。
  - 跟踪资源和城墙耐久。
  - 单例模式（Singleton）确保全局访问。
- **WaveManager**：
  - 控制僵尸生成和波次逻辑。
  - 使用 List 或 Queue 存储波次数据。
- **UnitController**：
  - 控制士兵和炮塔的攻击行为。
  - 使用 Transform.Find 或 Physics2D.OverlapCircleAll 寻找目标。
- **EnemyController**：
  - 控制僵尸移动（使用 MoveTowards）和攻击。
  - 包含 Animator 组件播放移动/攻击/死亡动画。
- **ProjectileController**：
  - 控制子弹/炮弹的移动和碰撞。
  - OnTriggerEnter2D 检测命中，应用伤害。
- **UIManager**：
  - 更新 UI 元素（资源、波次、城墙耐久）。
  - 处理按钮点击事件。

#### 3.3 物理与碰撞
- **碰撞体（Collider）**：
  - 士兵/炮塔：触发器，用于检测射程。
  - 僵尸/城墙：刚体（Rigidbody2D）+ 碰撞体，处理物理交互。
  - 子弹/炮弹：触发器，检测命中。
- **层（Layer）**：
  - 设置层（例如 Units、Enemies、Projectiles）优化碰撞检测。
  - 使用 LayerMask 限制射线检测范围。

#### 3.4 动画
- **Animator 组件**：
  - 士兵/炮塔：待机、射击动画。
  - 僵尸：移动、攻击、死亡动画。
- **触发条件**：
  - 使用 Animator 参数（例如 Bool 或 Trigger）切换状态。

#### 3.5 数据管理
- **ScriptableObject**：
  - 创建单位配置（士兵、炮塔、僵尸的属性）。
  - 创建波次配置（每波的僵尸类型和数量）。
- **保存系统**：
  - 使用 PlayerPrefs 或 JSON 保存进度（例如已解锁的波次、资源）。

### 4. 开发流程建议
1. **原型搭建**：
   - 创建基本场景：城墙、一个士兵、一个僵尸。
   - 实现简单的射击和移动逻辑。
2. **核心系统**：
   - 开发波次系统和资源管理。
   - 添加 UI 显示资源和波次。
3. **扩展功能**：
   - 增加多种僵尸和单位类型。
   - 实现升级和修复机制。
4. **优化与测试**：
   - 优化性能：对象池管理子弹/僵尸，减少 Instantiate/Destroy。
   - 测试平衡性：调整僵尸强度、资源获取速度。
5. **美化**：
   - 添加音效、粒子效果、动画。
   - 设计开始和失败界面。

### 5. 技术注意事项
- **性能优化**：
  - 使用对象池（Object Pooling）管理子弹和僵尸，减少运行时开销。
  - 限制同时存在的敌人数量，避免性能瓶颈。
- **可扩展性**：
  - 使用 ScriptableObject 存储配置，便于调整数值。
  - 设计模块化脚本，方便添加新单位或功能。
- **调试**：
  - 在 Inspector 中暴露关键变量（例如射程、射速），便于测试。
  - 使用 Debug.DrawRay 可视化射线检测。

### 6. 示例伪代码
以下是士兵射击逻辑的简单示例：

```csharp
public class SoldierController : MonoBehaviour
{
    public float fireRate = 1f; // 每秒射击次数
    public float range = 5f; // 射程
    public GameObject bulletPrefab;
    private float nextFireTime;

    void Update()
    {
        // 寻找最近的敌人
        GameObject target = FindClosestEnemy();
        if (target != null && Time.time >= nextFireTime)
        {
            Shoot(target);
            nextFireTime = Time.time + 1f / fireRate;
        }
    }

    GameObject FindClosestEnemy()
    {
        GameObject[] enemies = GameObject.FindGameObjectsWithTag("Enemy");
        GameObject closest = null;
        float minDistance = range;

        foreach (GameObject enemy in enemies)
        {
            float distance = Vector2.Distance(transform.position, enemy.transform.position);
            if (distance < minDistance)
            {
                minDistance = distance;
                closest = enemy;
            }
        }
        return closest;
    }

    void Shoot(GameObject target)
    {
        GameObject bullet = Instantiate(bulletPrefab, transform.position, Quaternion.identity);
        Vector2 direction = (target.transform.position - transform.position).normalized;
        bullet.GetComponent<Rigidbody2D>().velocity = direction * 10f; // 子弹速度
    }
}
```
